## рҹ“– What is Web3Varsity?

**Web3Varsity** is a comprehensive, gamified learn-to-earn Web3 education platform purpose-built on Hedera Hashgraph. It takes users from zero blockchain knowledge to confidently deploying smart contracts, minting NFTs, trading on DEXes, and understanding DeFi вҖ” all through structured, interactive courses with real on-chain activity on Hedera Testnet.

Unlike traditional e-learning platforms that rely on videos and static text, Web3Varsity delivers **hands-on blockchain education** where every lesson culminates in a verifiable on-chain action: sending HBAR, submitting HCS messages, deploying Solidity contracts, minting NFT badges, or earning on-chain certificates.

---

## рҹҺҜ Problem Statement

Web3 has an accessibility and education gap. The global blockchain education market is projected to reach **$18.6 billion by 2028**, yet millions of aspiring developers and non-technical enthusiasts вҖ” particularly across **Africa's 700M+ internet users under 30** вҖ” struggle to learn blockchain because:

1. **Existing education is fragmented** вҖ” YouTube videos, scattered docs, and text-heavy courses with no hands-on component.
2. **Blockchain interaction is intimidating** вҖ” New users are afraid to send transactions, deploy contracts, or interact with DeFi protocols for fear of losing real money.
3. **No credentialing standard** вҖ” Traditional certificates (PDFs, emails) are easily forged and carry no verifiable trust.
4. **The tooling learning curve is steep** вҖ” Setting up wallets, networks, and development environments creates friction before learning even begins.

## рҹ”‘ Why Web3? Why Not Web2?

This solution **requires blockchain** вҖ” a Web2 alternative cannot replicate its core value:

1. **Unforgeable credentials** вҖ” NFT certificates on HTS are cryptographically verifiable by anyone. PDFs and emails are trivially forged.
2. **User sovereignty** вҖ” Wallet-based authentication means users own their identity and credentials without depending on a centralized provider.
3. **Censorship-resistant community** вҖ” HCS message board cannot be censored, edited, or deleted by any single party.
4. **Real blockchain interaction** вҖ” Users sending real HBAR, deploying real contracts, and minting real NFTs on testnet builds genuine confidence. No Web2 simulation can replicate the experience of interacting with an actual distributed ledger.
5. **Composable credentials** вҖ” On-chain NFT certificates are portable across wallets, platforms, and even chains вҖ” they outlive any single platform.

## рҹ”— Deep Hedera Integration

Web3Varsity leverages **5 core Hedera network services**, creating one of the deepest integrations in the ecosystem:

### 1. Hedera Token Service (HTS)
- **NFT Certificate Collection** вҖ” Completion certificates are minted as NFTs on HTS with custom metadata
- **NFT Badge System** вҖ” Achievement badges (Common вҶ’ Legendary) minted on-chain when earned
- **Token Association** вҖ” Users associate with token collections before receiving NFTs
- `src/lib/hedera/hts-service.ts` В· `src/lib/hedera/nft-certificates.ts`

### 2. Hedera Consensus Service (HCS)
- **Community Message Board** вҖ” Real-time decentralized messaging via HCS Topic `0.0.7180075`
- **On-Chain Activity Log** вҖ” All community messages are immutably recorded with consensus timestamps
- **Topic Subscription** вҖ” Polling-based real-time message feed from Mirror Node
- `src/lib/hedera/hcs-service.ts` В· `src/components/course/practical/HCSMessageBoard.tsx`

### 3. Hedera Smart Contract Service (HSCS)
- **Smart Contract Playground** вҖ” Deploy real Solidity contracts (Counter, MessageStorage, Voting) to Hedera testnet
- **Contract Interaction** вҖ” Execute functions and query state on deployed contracts
- **Bytecode Compilation** вҖ” Pre-compiled bytecodes using `solc 0.8.20` with London EVM compatibility
- `src/lib/hedera/contracts-service.ts` В· `compile-contracts.cjs`

### 4. Hedera File Service (HFS)
- **Certificate SVG Storage** вҖ” Generated SVG certificate images uploaded to HFS (a creative, novel use of this typically underutilized service)
- **Metadata JSON Storage** вҖ” NFT metadata stored on-chain with chunked uploads for files >4KB
- **Mirror Node Retrieval** вҖ” Files fetched for verification without HBAR cost
- `src/lib/hedera/certificate-generator.ts`

### 5. HBAR Transactions
- **Send HBAR Lessons** вҖ” Users send real testnet HBAR in practical lessons
- **In-Course Faucet** вҖ” A purpose-built testnet faucet (`request-faucet` Edge Function) is embedded directly inside practical lesson components (e.g., WalletCreator). When a student reaches a step that requires HBAR, they click "Get Testnet HBAR" without leaving the lesson. Rate-limited to 10 HBAR/day per user with full transaction tracking in a dedicated `faucet_requests` database table.
- **Standalone Faucet Page** вҖ” Dedicated `/faucet` page with eligibility checks, cooldown timers, amount selection, and complete faucet history
- **Transaction History** вҖ” Full history queryable via Mirror Node API with type filtering (send, receive, faucet)
- `src/lib/hedera/transactions.ts` В· `src/lib/hedera/faucet.ts` В· `supabase/functions/request-faucet/`

### Ecosystem Partners Used
- **Bonzo Finance** вҖ” DeFi simulator with Bonzo vault yield optimization strategies and Keeper Agent logic
- **Hashgraph Online (HOL)** вҖ” AI Tutor registered as a discoverable agent on the HOL Registry Broker via HCS-10 protocol
- **Hedera Wallet Connect** вҖ” Native wallet connectivity for HashPack, Blade, and other Hedera-native wallets
- **Hedera Mirror Node** вҖ” Account lookups, balance queries, transaction verification, NFT metadata retrieval

- **Native Hedera Wallets** вҖ” HashPack, Blade via EIP-6963 discovery and WalletConnect v2 fallback
- **Unified Provider Interface** вҖ” Abstraction layer (`IWalletProvider`) normalizes EVM and native Hedera signing
- **Signature-Based Auth** вҖ” Passwordless login: sign a message вҶ’ Supabase JWT issued

### рҹ’§ Integrated Testnet Faucet System
- **In-Course Funding** вҖ” The faucet is embedded directly inside practical lesson components (`WalletCreator`). When a student creates their first wallet, they can instantly request testnet HBAR without navigating away вҖ” eliminating the #1 friction point in blockchain education ("how do I get test tokens?")
- **Standalone Faucet Page** вҖ” Dedicated `/faucet` page with configurable amount selection, eligibility checks, cooldown timers, and transaction history
- **Rate-Limited via Edge Function** вҖ” The `request-faucet` Supabase Edge Function enforces per-user daily limits (10 HBAR/day) and cooldown periods (24 hours). All requests are logged to the `faucet_requests` table for admin analytics
- **Mirror Node Verification** вҖ” After faucet distribution, the component queries the Hedera Mirror Node to verify the new balance in real time, teaching students how on-chain state confirmation works
- **Admin Dashboard Integration** вҖ” Faucet statistics (total distributed, unique users, daily usage) are visible in the admin dashboard for platform monitoring

- **Multi-Step Wizard** вҖ” 5-step course creation: Metadata вҶ’ Objectives вҶ’ Lessons вҶ’ Preview вҶ’ Publish
- **Lesson Editor** вҖ” Rich editor for text, interactive components, quizzes, and practical labs
- **Quality Monitor** вҖ” Real-time validation scoring for course completeness
- **Admin Review** вҖ” Courses go through approval workflow before publishing

## рҹҸ—пёҸ Key Design Decisions

| Decision | Rationale |
|---|---|
| **Supabase over custom backend** | Rapid iteration with built-in Auth, PostgreSQL, and Edge Functions. Row-Level Security (RLS) enforces user-scoped data access without custom middleware. Allowed a solo developer to build a production-grade backend in weeks, not months. |
| **Dual wallet architecture (EVM + Native Hedera)** | Maximizes user reach. MetaMask users (the largest Web3 wallet base) can connect immediately, while HashPack/Blade users get a first-class native experience. The `IWalletProvider` abstraction normalizes signing across both. |
| **Zustand over Redux for state management** | Simpler API with less boilerplate for the course creation wizard. Zustand's `persist` middleware gives free localStorage draft saving. React Query handles all server state, so Zustand only manages local UI state. |
| **Server-side Hedera operator keys** | All Hedera private keys are stored exclusively in Supabase Edge Function secrets вҖ” never exposed to the client. This prevents key extraction from browser dev tools and follows security best practices. |
| **HFS for certificate storage (not IPFS-only)** | HFS keeps certificate images fully on Hedera's network вҖ” no external dependency on IPFS pinning services. IPFS/Pinata is used as a fallback for redundancy. |
| **AI via Supabase Edge Functions (not client-side)** | Gemini API keys are kept server-side. Rate limiting (5 generations/hour, 20 chats/hour) is enforced in the Edge Function, preventing abuse. |
| **Solidity pre-compilation** | Educational contracts are pre-compiled to bytecode using `solc 0.8.20` with London EVM compatibility. This removes the need for users to install Solidity toolchain and ensures consistent deployments. |

## рҹҡҖ Getting Started

### Prerequisites

- **Node.js** >= 18
- **pnpm** (package manager)
- **MetaMask** or **HashPack** browser extension
- A Hedera Testnet account ([portal.hedera.com](https://portal.hedera.com))

### Installation

```bash
# Clone the repository
git clone https://github.com/jtmax1/Web3Varsity.git
cd Web3Varsity

# Install dependencies
pnpm install

# Configure environment variables
cp .env.example .env
# Edit .env with your Supabase and Hedera credentials
```

### Environment Variables

```env
# Supabase Configuration
VITE_SUPABASE_URL=your_supabase_project_url
VITE_SUPABASE_ANON_KEY=your_supabase_anon_key

# Hedera Network Configuration
VITE_HEDERA_NETWORK=testnet
VITE_HEDERA_MIRROR_NODE=https://testnet.mirrornode.hedera.com
VITE_HEDERA_RPC_URL=https://testnet.hashio.io/api

# Hedera Agent Kit Configuration
VITE_OPENAI_API_KEY=your_openai_key
VITE_AGENT_ACCOUNT_ID=your_agent_account_id
VITE_AGENT_PRIVATE_KEY=your_agent_private_key

# Protocol Configurations
VITE_BONZO_TREASURY=0.0.12345
VITE_BONZO_MARKET_ID=your_market_id
```

### Running Locally

```bash
# Start development server (http://localhost:3000)
pnpm dev

# Build for production
pnpm build

# Start MCP server (optional, for AI agent integration)
npx tsx backend/mcp-server.ts
```

### Compiling Smart Contracts

```bash
# Compile educational Solidity contracts to bytecodes
node compile-contracts.cjs
```
